Types of Searches

Searching through long lists or arrays has always caused headaches to new programmers. There are mainly two ways by which we can classify our methods to search a particular item in a long list. These searches can be best depicted by a common game we might have played as children: the guessing game. Consider that you are playing a game with your friend - you tell your friend to guess a number that you are thinking about and give him a range (for example, 0 to 100). With each guess, you tell him/her if her guess is too high or too low. If the guess is correct, you tell him to stop.

Simple Search Binary Search
  • The simple search is a method of searching an element in the array right from the beginning. The search starts from the first position (index 0) and keeps shifting by one position till the required element is found.
  • Thus, considering the above given example, the simple search starts from 0 and ends at 100 (till the element is found).
  • The binary search starts at the arithmetic mean of the range. Considering the above example, the search starts at 50.
  • Then, a guess is made. If the guess is too low, the range (50-100) is considered. Else, the range (0-50) is considered and the same algorithm is repeated.


Algorithm running times grow at different rates

The binary search witnesses a low-slope increment in the number of functions it carries as well as the time as the number of items in the array increases. Thus, binary search provides a fast result. Simple search, on the other hand, has a steeper slope when it comes to the number of tasks carried out. Thus, it takes more time than binary search.
