Stacking

Written By Ved Panse Last Modified: 30 June 2021

In this section of the documentation, we will be looking at stacking: an efficient program that is known by many. It has also made its way into job interview questions asked by Big Tech companies around the world.

Let's do this!

What is Stacking?

Stacking is a common terminology when you learn to program. It can be best explained with an analogy. Imagine that you are having a stack of playing cards with you. The elements in a list and arrays are stored in the same way. There are two functions that you can perform with the given lists and arrays.



Push Pop
It is the same as appending to the end of the list. For example, adding a card to the stack of cards. It is reading the topmost element and then removing it. For example, this function reads the top-most card in the stack and then pops it off the stack.


How is this related to our topic?

Well, it turns out that the functions which we write work in a similar manner. They get stacked one over the other and are perceived by the computer in this way.



Why is Stack Algorithm necessary to be known?

With stack, you do not have to keep a check on the functions that have been completed. The stack function does it for you. Using stack is convenient but there is a cost: saving all that info can take up a lot of memory. Each of those function calls takes up some memory. At that point, you may choose any of the following courses:

  1. Rewrite the code to use a loop instead.
  2. Use tail recursion.
