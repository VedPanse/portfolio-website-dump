The Big O Notation

Written By Ved Panse Last Modified: 30 June 2021

In this section of the documentation, we will be looking at a concept that you might have never heard about. The Big O Notation is also not covered by any professional courses that teach you programming languages. Though this might be a new concept and may seem to be complex, it is actually very easy.

The Big O

The Big O notation is a simple method that tells you how fast a computer program runs. However, it doesn't measure the time taken by the algorithm. It denotes the number of tasks that that particular algorithm takes.

The following table depicts the run times for simple search and binary search.



Simple Search Binary Search
100 items → 100 guesses 100 items → 7 guesses
4,000,000,000 items → 4,000,000,000 guesses 4,000,000,000 items → 32 guesses
O(n)...Linear Time O(log n)...Logarithmic Time


This table measures the maximum possible guesses it may take.

As you can see, binary search takes very few guesses, thus making it far more efficient than simple search. However, everything has its own disadvantages. As binary search follows a complicated algorithm, it increases the chances of bugs and errors. Simple search, after iterating over the complete array, reduces the chances of any such bugs and errors to a negligible extent.



Applications and need of such techniques

Imagine yourself writing a program for social media websites. You have been assigned to write a program that goes through an array of all users and provides the users relevant data of the value they input in a search box. This is where your search algorithm comes into play. You must consider that you are working with arrays with billions of elements in them. Thus, the speed should be fast and with minimum code.



Some Common Big O Run times

From fastest to slowest

  1. O (log n), also known as log time. Example: Binary search.
  2. O(n), also known as linear time. Example: Simple search.
  3. O(n*log n). Example: A fast sorting algorithm, like quick-sort.
  4. O(n^2). Example: A slow sorting algorithm, like selection sort.
  5. O(n!). Example: A really slow algorithm, like the traveling salesperson.
