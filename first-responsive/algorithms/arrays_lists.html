Algorithms for Arrays and Lists

Written By Ved Panse Last Modified: 30 June 2021

In this section of the documentation, we will be looking at algorithms for arrays and lists. This section gives you a sneek peak into when and where are arrays and lists are used. Also, we will be relating their properties such as append and deletion to compare.

Let's get going!

Appendage and Insertions

Appending elements means adding elements to the last of a pre-existing list. This activity is seen in nearly all the programs that manage a database. Insertions means inserting an element in any position in an array/list.



Lists Arrays
With the availability of free space anywhere in the computer (as mentioned in How Are Items Stored in a Computer), the newly appended elements can be arranged anywhere. Considering the general example, books can be put in any available slot, it is that easy. The same doesn't hold true for arrays. If consecutive space is not available to store the element(s) to be appended, the entire array has to be shifted to some other place. Thus, the position of every element has to be shifted.


Conclusion: Lists are better than arrays in this respect.



Deletion

Deletion is the process of deleting an element from an array/list. While appendage is not always possible (due to insufficient memory or other conditions), deletion of any element in the list/array is always possible.



Lists Arrays
By deleting an element in a list, you just change what the previous elements point to. Thus, it becomes time and memory-efficient. By deleting an element in an array, the entire array has to be moved up by one space. This involves a lot of processing and leads to a slow code when working with arrays containing billions of elements.


Conclusion: Lists are better than arrays in this respect.



Comparison Of Usage Frequency Of Lists And Arrays

Usage of these datatypes heavily depends on the nature of the project you are working on. Arrays can store data very compactly and are more efficient for storing large amounts of data. Arrays are great for numerical operations; lists cannot directly handle math operations. For example, you can divide each element of an array by the same number with just one line of code. However, lists are faster.

Arrays see a lot of use. This is because it functions on the fundamentals of random access. On the other hand, linked lists function on sequential access.



Sequential Access Random Access
This process accesses the required element beginning from the start. Thinking of it in terms of bookshelf and books, it finds the required book after going through the entire bookshelf at least once and then pointing that book out. This process may begin at any position. It may directly jump to the eighth position, saving us a lot of time.
