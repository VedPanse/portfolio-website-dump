Clustering

Written By Ved Panse Last Modified: 20 May 2022

K-Means Clustering

K-Means is an example of unsupervised learning. Thus, it makes inferences and does not dwell on historical data. Clustering intends to group data on several factors and discover the patterns to make predictions. For this, it requires to decide a fixed number of clusters. Without this, the algorithm could form any number of clusters: it can even create a separate cluster for every data point. No one would want that.

Image from https://www.geeksforgeeks.org/ml-k-means-algorithm/

Let the number of clusters be k. Thus, we will plot the k data points. These data points (highlighted in black) are known as centroids. Then, other data points will be assigned to the centroids that are closest to them. All the data points with the same centroid will be in the same cluster.

Hierarchical Clustering

Hierarchical clustering forms a hierarchial order of data - starting from the dominant data at the top. It consists of two types:-

Agglomerative Clustering

As the name states, agglomerative clustering forms agglomerates of clusters. In other words, the algorithm combines clusters. The following image is probably enough to explain.

Divisive Hierarchical Clustering is the opposite of Agglomerative Clustering.
