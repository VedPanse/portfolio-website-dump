How are items stored in a computer?

Imagine a computer to be a bookshelf that is already filled with some books, but is not full. The books that already occupy the bookshelf are kept in a random order and not next to each other. Consider the items that you want to be stored in the bookshelf to be books. Further explanation has been split into two datatypes: arrays and lists to understand two different cases of memory storage.

Arrays

In arrays, their elements are stored one after the other on the shelf, just like books, in the order specified. Take, for example, that there is a shelf and you have 10 books to be stored. But, the maximum capacity for accommodating it in the bookshelf one after the other is seven (and the eighth position is occupied by something else). The books cannot be stored in an array, thus leading to problems dealing with memory. The elements cannot be spread over the place, like in the case of lists.

Lists

In lists, their elements are stored in order, but not necessarily one after the other. The elements are spread over to occupy any given space. Take for example that there are 10 books to be arranged. But, the maximum capacity for accommodating it in the bookshelf one after the other is seven (and the eighth position is occupied by something else). In lists, the first seven elements get stored in the order provided, while the other elements occur after the seventh position. One element stores the link of the next one.



Types of Searches

Searching through long lists or arrays has always caused headaches to new programmers. There are mainly two ways by which we can classify our methods to search a particular item in a long list. These searches can be best depicted by a common game we might have played as children: the guessing game. Consider that you are playing a game with your friend - you tell your friend to guess a number that you are thinking about and give him a range (for example, 0 to 100). With each guess, you tell him/her if her guess is too high or too low. If the guess is correct, you tell him to stop.

Simple Search

The simple search is a method of searching an element in the array right from the beginning. The search starts from the first position (index 0) and keeps shifting by one position till the required element is found. Thus, considering the above given example, the simple search starts from 0 and ends at 100 (till the element is found).

Binary Search

The binary search starts at the arithmetic mean of the range. Considering the above example, the search starts at 50. Then, a guess is made. If the guess is too low, the range (50-100) is considered. Else, the range (0-50) is considered and the same algorithm is repeated.

Algorithm running times grow at different rates

The binary search witnesses a low-slope increment in the number of functions it carries as well as the time as the number of items in the array increases. Thus, binary search provides a fast result. Simple search, on the other hand, has a steeper slope when it comes to the number of tasks carried out. Thus, it takes more time than binary search.

The Big O

The Big O notation is a simple method that tells you how fast a computer program runs. However, it doesn't measure the time taken by the algorithm. It denotes the number of tasks that that particular algorithm takes.

The following table depicts the run times for simple search and binary search.


Simple Search Binary Search

4,000,000,000 items —> 4,000,000,000 guesses 4,000,000,000 items -> 32 guesses

This table measures the maximum possible guesses it may take.

As you can see, binary search takes very few guesses, thus making it far more efficient than simple search. However, everything has its own disadvantages. As binary search follows a complicated algorithm, it increases the chances of bugs and errors. Simple search, after iterating over the complete array, reduces the chances of any such bugs and errors to a negligible extent.

Applications and need of such techniques

Imagine yourself writing a program for social media websites. You have been assigned to write a program that goes through an array of all users and provides the users relevant data of the value they input in a search box. This is where your search algorithm comes into play. You must consider that you are working with arrays with billions of elements in them. Thus, the speed should be fast and with minimum code.

Some Common Big O run times

From fastest to slowest

  1. O (log n), also known as log time. Example: Binary search.
  2. O(n), also known as linear time. Example: Simple search.
  3. O(n*log n). Example: A fast sorting algorithm, like quick-sort.
  4. O(n^2). Example: A slow sorting algorithm, like selection sort.
  5. O(n!). Example: A really slow algorithm, like the traveling salesperson.



Algorithms for Arrays and Lists

Appendage and Insertions

Appending elements means adding elements to the last of a pre-existing list. This activity is seen in nearly all the programs that manage a database.
Insertions means inserting an element in any position in an array/list.

Lists: With the availability of free space anywhere in the computer (as mentioned in How Are Items Stored in a Computer), the newly appended elements can be arranged anywhere. Considering the general example, books can be put in any available slot, it is that easy.

Arrays: The same doesn't hold true for arrays. If consecutive space is not available to store the element(s) to be appended, the entire array has to be shifted to some other place. Thus, the position of every element has to be shifted.

Conclusion: Lists are better than arrays in this respect.

Deletion

Deletion is the process of deleting an element from an array/list. While appendage is not always possible (due to insufficient memory or other conditions), deletion of any element in the list/array is always possible.

Lists: By deleting an element in a list, you just change what the previous elements point to. Thus, it becomes time and memory-efficient.

Arrays: By deleting an element in an array, the entire array has to be moved up by one space. This involves a lot of processing and leads to a slow code when working with arrays containing billions of elements.

Conclusion: Thus, lists are definitely better than arrays when it comes to deletion.

Comparison of usage frequency of lists and arrays

Usage of these datatypes heavily depends on the nature of the project you are working on. Arrays can store data very compactly and are more efficient for storing large amounts of data. Arrays are great for numerical operations; lists cannot directly handle math operations. For example, you can divide each element of an array by the same number with just one line of code. However, lists are faster.
Arrays see a lot of use. This is because it functions on the fundamentals of random access. On the other hand, linked lists function on sequential access.

Sequential Access: This process accesses the required element beginning from the start. Thinking of it in terms of bookshelf and books, it finds the required book after going through the entire bookshelf at least once and then pointing that book out.

Random Access: This process may begin at any position. It may directly jump to the eighth position, saving us a lot of time.



Recursion

What are recursive programs?

Recursive programs are those which keep calling themselves until they are told to stop. If not stopped, they can perform infinite loops.

def fact(x): #fact -> factorial
    if x == 1:
        return 1
    else:
        return x * fact(x-1)

print(fact(12))

Cases in recursive code

Recursive Case: This is the case when the function calls itself and thus, leads to an infinite loop.

Base Case: This is the code that is added to the code block to prevent recursions.



Stacking

Stacking is a common terminology when you learn to program. It can be best explained with an analogy.
Imagine that you are having a stack of playing cards with you. The elements in a list and arrays are stored in the same way. There are two functions that you can perform with the given lists and arrays.

Push

It is the same as appending to the end of the list. For example, adding a card to the stack of cards.

Pop

It is reading the topmost element and then removing it. For example, this function reads the top-most card in the stack and then pops it off the stack.

How is this related to our topic?

Well, it turns out that the functions that we write work in a similar manner. They get stacked one over the other and are perceived by the computer in this way.

Why is Stack Algorithm necessary to be known?

With stack, you do not have to keep a check on the functions that have been completed. The stack function does it for you.
Using stack is convenient but there is a cost: saving all that info can take up a lot of memory. Each of those function calls takes up some memory. At that point, you may choose any of the following courses:

  1. Rewrite the code to use a loop instead.
  2. Use tail recursion.

Both are equally effective and widely used. Tail recursion is supported by some languages, but not all of them. Thus, the first course is suggested to new programmers.
